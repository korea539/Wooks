데이터 직무의 중요성

데이터분석 열풍

IT 업계뿐만 아니라 제조, 생산, 서비스, 유통업 등 다양한 분야에서 데이터의 중요성이 부각되고 있습니다. 몇몇 기업에서 데이터 분석을 도입하여 성공을 거둔 사례들이 등장했기 때문입니다. 이에 따라 많은 전통적 기업들에서도 디지털 트랜스포메이션(디지털 전환)* 열풍이 불고 있습니다. 데이터 분석의 중요성과 필요성을 느꼈기 때문입니다.

데이터 관련 직무

기업들이 데이터 활용에 중요성을 느끼는 만큼 각 기업에서 데이터 관련 직무가 생기는 추세입니다. 이제 막 생성되는 직무인 만큼 회사마다 직무의 역할이 다를 수 있지만, 회사가 데이터 전문가들에게 원하는 것을 명확합니다. ‘의사결정자가 활용할 수 있도록 데이터를 유의미하게 분석해 주는 것.’입니다.
데이터를 유의미하게 분석한다는 것에는 많은 기술들이 함축되어 있습니다.

일단 수없이 쏟아지는 데이터를 저장하고 꺼내올 수 있는 기술이 필요합니다. 그리고 날것의 데이터를 처리 할 수 있는 기술과 분석 할 수 있는 기술, 이를 설명하고 납득시키기 위해 표현하는 기술, 더 나아가 데이터의 패턴을 찾고 예측할 수 있는 기술들이 있습니다.
이렇게 다양한 전문 기술이 사용되는 만큼 데이터 직무는 고도화된 분야입니다. 그만큼 여러 전문가들이 필요합니다. 직무로는 크게 세 가지로 나누어 볼 수 있습니다.

데이터 직무 시작하는 방법

시작하는 방법은 비전공자들 기준으로 말씀드리겠습니다. 위의 세 직무에서 대체로 공통으로 요구되는 사항이자, 모든 교육과정의 시작에 배우는 것이 있습니다. 바로 파이썬입니다.

데이터 엔지니어의 경우 회사의 따라 Java를 쓰기도 하지만, Java를 알기 전 좀 더 직관적인 언어인 파이썬으로 입문한다면 추후 Java 도 어렵지 않게 학습할 수 있습니다.

데이터 분석가의 경우에도 회사에 따라 SQL만 다루는 경우도 있지만, 대체로 데이터를 다루고 처리하기 때문에 파이썬이 필요한 경우가 많습니다. 파이썬 이후 판다스까지 같이 학습하는 걸 추천드립니다.

데이터 사이언티스트의 경우 주로 ML/DL 이 업무의 주를 이루지만 머신러닝을 하기 위해서 tensorflow 나 pytorch를 사용하는데 이들의 베이스 언어가 파이썬입니다.

일단 모든 분야의 기본 언어가 될 수 있는 파이썬을 공부해 보시기 시작하는걸 추천드립니다. 시작인 반이기 때문에 파이썬 공부를 시작한 것만으로도 이미 가장 어려운 반을 하신 거라고 생각합니다.
파이썬은 개발 언어로 사용되기 때문에 데이터 업무를 하기 위해서는 파이썬의 모든 부분을 다 알아야 할 필요는 없습니다. 데이터 분석을 목적으로 파이썬을 사용할 때 어떤 부분이 필요한지, 파이썬 목차에서 짚어드리도록 하겠습니다

부트캠프를 듣는 게 좋을까?

데이터 직무를 하기 위해 꼭 부트캠프를 들어야 하는 것은 아닙니다. 데이터 분석 관련 학회를 들거나, 동아리, 스터디 등 다양한 학습 통로가 있습니다. 다른 방법에 비해 부트캠프가 가진 장점은 단기간에 몰입해서 학습을 할 수 있다는 점입니다. 그리고 취업에 특화되어 있습니다. 그렇기 때문에 데이터 직무에 뜻이 있는 비전공자 분들은 부트캠프를 듣는 걸 추천드립니다. 다만 부트캠프를 선택하기 전에 , 파이썬이란 어떤 것인지 한 번이라도 공부해 보시길 바랍니다.

부트캠프 고르는 팁

그리고 부트캠프를 고를 때, 커리큘럼과 관련 후기를 많이 알아보시는 것이 좋습니다. 특히 크고 작은 프로젝트를 많이 하는 부트캠프는 교육 외 시간이 많이 소요되고, 체력적으로 힘들 수 있는 반면, 배운 기술을 곧바로 연습하기 좋다는 장점이 있습니다. 같은 직무에 대한 부트캠프여도 캠프마다 커리큘럼이 조금씩 다릅니다. 본인이 원하는 학습내용이 들어가 있는지도 확인해 보시길 바랍니다.

데이터 직무랑 나랑 잘 맞는지 어떻게 알 수 있을까?

이 직무가 나와 적성이 잘 맞는지 확인할 길은 직접 해보는 게 가장 정확한 방법이 아닐까요. 여러분이 ‘데이터 분석을 해볼까?’라는 생각을 떠올렸다면, 마음속에 내재된 데이터에 대한 관심과 호감이 있는 분이 분명합니다. 평소에도 사실 기반으로 말하는 것을 좋아하고 팩트체크에 진심인 분, 데이터 기반 판단을 좋아하시는 분이라면 데이터 직무가 잘 맞을 것이라 예상됩니다. 데이터 직무에는 위에 언급한 데이터를 직접 기술적으로 다루는 직무 이외에도 데이터를 활용하는 그로스 해커나, 데이터 마케터 등도 있기 때문에 다양한 직무들을 확인해 보시는 걸 추천드립니다.

‘데이터 분석' 부트캠프 커리큘럼 비교

이번에는 실제 부트캠프 커리큘럼들을 함께 살펴보며 조금 더 구체적인 이야기를 해보겠습니다. 부트캠프는 ‘AI 부트캠프’라는 이름으로 데이터 직군 전반에 사용될 수 있는 지식을 배우는 경우가 많습니다. 하지만 보다 직관적인 비교를 위해 ‘데이터분석’만을 다루는 부트캠프 커리큘럼 3개를 예시로 선정하여 각각 비교해 보겠습니다.

세 부트캠프의 공통점은 파이썬을 배우는 비중이 상당히 크다는 것입니다. 파이썬을 활용하여 데이터 분석을 할 줄 아는 것은 데이터 분석가에게 가장 기본적으로 요구되는 사항이기 때문입니다. 이밖에도 파이썬으로 하는 웹크롤링 등 데이터 수집에 관한 내용을 배우는 곳도 있습니다.
한편 SQL과 Tableau는 각 커리큘럼의 제목에서도 볼 수 있듯이 배우는 목적이나 중요도가 다르다는 것을 알 수 있습니다. 파이썬에 비해 매우 적은 시간을 투입하는 special lecture 형식으로 구성하는 부트캠프도 있는 반면, 태블로를 기초, 심화, 실전으로 나누어 길게 배우는 부트캠프도 있습니다.

이렇게 특정 내용에 대한 비중이 다른 경우 어떤 기준으로 커리큘럼을 선택할 수 있을까요? 데이터분석가에게 어떤 기술이나 역량을 요구하는지 서술된 각 회사의 JD(직무 기술서)를 살펴보는 방법이 있습니다.
간혹 데이터 분석가 JD 중에는 파이썬에 대한 언급 없이 SQL 쿼리를 짜는 능력만을 요구하거나, 대시보드 제작에 초점을 맞춘 JD도 존재합니다. 따라서 JD를 먼저 분석하고 자신이 입사하고 싶은 회사에서는 어떤 기술을 요구하는지 정리한 뒤 가장 적합한 부트캠프를 선택하는 것이 필요합니다.

통계 지식은 데이터분석 부트캠프에서 직접적으로 학습하는 비중이 크지 않은 편이지만 실제 JD에서는 통계학 전공자를 우대하거나 기초 통계 지식을 필수적으로 요구하는 사례가 많습니다. 물론 데이터 분석 툴을 다룰 줄 아는 것도 중요하지만, 어떤 사고과정과 기준을 바탕으로 툴을 사용했는지가 중요하기 때문입니다. 그러한 기준의 객관적인 근거가 통계 지식이 될 수 있기 때문에 이에 대한 학습이 필요합니다.
딥러닝의 경우 ‘데이터 분석’ 부트캠프에서는 다루지 않거나 기본 개념을 이해하는 정도에 그치는 경우가 대부분입니다. 반면 머신러닝은 데이터 분석 부트캠프에서 빠지지 않고 다루는 만큼 그 중요도가 꽤나 높다고 볼 수 있습니다. 물론 회사에서 머신러닝을 다룰 줄 알아야 한다고 명시하는 경우는 많지 않지만, 비즈니스 데이터를 분석하거나 예측할 때 유용하게 사용될 수 있습니다.
‘기타’ 분류에 있는 항목들은 데이터분석가 JD에서 요구하지는 않지만 데이터 직군 내에서 직무 변경을 하거나 직무 자체가 모호한 경우 활용될 수 있습니다. 따라서 데이터 사이언티스트나 데이터 엔지니어 등의 데이터 직군에도 흥미가 있는지 스스로 검토해본 후 ‘기타’ 비중에 대한 판단을 내리면 될 것입니다.

이렇게 실제 부트캠프 커리큘럼을 살펴보는 과정에서 ‘JD’라는 낯선 단어가 많이 등장했음을 알 수 있습니다. 부트캠프의 커리큘럼은 JD에서 요구하는 자격조건을 충족시키기 위해 구성되는 경우가 많습니다. 그리고 실제 취업을 준비할 때에도 JD 자체가 하나의 가이드가 될 수 있습니다. 그렇다면 JD가 무엇이고, 왜 중요한지 구체적으로 알아보겠습니다.

JD (직무 기술서)

Job description 이란?

JD(Job Description)란 직무 기술서를 의미합니다. 전통적으로 직무 기술서(Job Description)에는 ‘과업’에 초점을 맞추어 직무의 내용이나 수행과정을 상세히 기술했습니다. 반면, 직무 명세서(Job Specification)에는 ‘사람’에 초점을 맞추어 직무 관련 자격요건이나 필요한 가치관 등을 적었습니다.
하지만 실제 인재 선발, 즉 채용 단계에서의 직무 기술서는 ‘과업’과 ‘사람’에 대한 설명이 모두 포함된 경우가 많습니다. 보통 [회사/ 팀에 대한 소개- 업무에 대한 소개- 자격 요건- 우대 사항- 채용 프로세스- 복리후생- 고용 형태- 회사 위치] 의 정보를 담고 있습니다.

JD의 중요성

직무 기술서, 즉 JD는 회사 입장에서 가장 중시하는 가치와 선발 기준을 드러내는 것입니다. 따라서 먼저 JD에 간단히 드러난 회사의 가치관이나 인재상을 보고 본인과의 일치 여부를 따져보아야 합니다. 일치한다면, JD에서 요구하는 사항들을 최대한 녹여서 효과적으로 자기소개서를 작성하고 면접을 준비할 수 있습니다. 또한 많은 직무 기술서는 필수적으로 갖춰야 하는 ‘자격요건’과, 갖추면 좋은 ‘우대사항’을 구분하여 보여줍니다. 따라서 이를 바탕으로 우선순위를 세운다면 보다 효율적인 공부와 취업 준비가 가능해집니다.

아래는 본 책에서 주로 다루는 직종인 ‘데이터분석가’의 직무 기술서 예시입니다. 한 대형 유통업체의 JD 내용을 바탕으로 보편적으로 데이터분석가에게 요구되는 내용들을 담았습니다.

외국의 경우 일반화 할 수는 없지만 복지를 강조하기 보다는 아래 예시와 같이 회사나 직무에 대한 소개와 자격 요건을 중점으로 작성된 경우가 많습니다. 또한 우리나라의 직무기술서에는 대부분 급여정보가 포함되지 않은 반면, 외국은 full-time이 아닌 계약직 등은 시간당 급여를, full-time은 월 급여나 연봉의 범위를 기술한다는 것이 대조적입니다.

JD로 살펴본 현실 데이터 직무

현실에서의 데이터 직무는 데이터 분석가, 데이터 사이언티스트, 데이터 엔지니어의 구분이 명확하게 이루어지는 경우도, 그 경계가 모호한 경우도 있습니다. 예를 들어 첫번째 그림의 회사에서는 데이터 분석가의 가장 기본이 되는 업무만을 기술하였습니다. 데이터 분석가가 SQL을 활용한 분석과 인사이트 도출을 위한 대시보드 제작을 한다고 명시하였고, 자격 요건도 많지 않습니다. 반면, 두번째 그림과 같이 데이터 분석가가 데이터수집 시스템과 통계 시스템을 직접 개발하는 회사도 있습니다. 이 경우 데이터 분석가에게 데이터 파이프라인 설계/구축/운영 경험이나 분산 처리 시스템 활용 개발 경험을 요구하기도 합니다.

심지어 우리나라에는 데이터 분석가, 데이터 사이언티스트, 데이터 엔지니어 외에도 데이터 리서처, 머신러닝 엔지니어, 그로스 해커 등 무수히 많은 이름의 데이터 전문가가 존재하여 혼란을 주기도 합니다.
하지만 저희는 가장 대표적인 세 가지 직군의 특징을 먼저 살펴보기로 하였습니다. 먼저 보편적인 기준이 정립된 상태로 추후 다른 직군들과의 차이점이나, 실제 현장에서 업무가 합쳐질 수 있는 경우를 고민하는 것이 바람직하기 때문입니다.
따라서 유명 IT 기업 7곳(네카라쿠배당토)의 평균적인 JD 내용을 바탕으로 대표 데이터 직군에서 하는 일과 요구 역량, 우대사항에 대해 각각 살펴보겠습니다. 먼저 데이터 분석가입니다.

데이터 분석가

데이터 분석가는 이름 그대로 데이터 분석 도구를 활용하여 데이터를 분석한 뒤 시각화를 진행하고 인사이트를 제공하는 역할을 합니다. 이때의 ‘인사이트’는 회사가 직면한 문제와 이를 해결할 수 있는 방안을 의미합니다. 회사나 팀의 전략 수립을 위한 데이터를 분석해야 하므로 경영진이나 기획자, 개발자와의 원활한 커뮤니케이션 능력이 필요합니다. 직무 기술서에서 요구하는 기술 역량의 범위에 따라 단순한 데이터 분석 외에 모델링이나 데이터 분석 환경 개선 등을 하기도 합니다. 7곳의 유명 IT의 직무기술서 모두 통계 및 데이터 분석 방법론에 대한 이해를 강조하였다는 것이 특징적입니다.

데이터 사이언티스트

데이터 사이언티스트는 데이터 과학자라고도 불립니다. 고객 행동 패턴에 대한 모델링을 진행하거나 예측 모델링, 추천 시스템 등을 개발하여 비즈니스 인사이트를 제공을 하는 등 데이터 분석 뿐만아니라, 통계, 머신러닝, 딥러닝 등이 활용되는 예측, 추론 모델을 설계합니다. 이를 통해 데이터 사이언티스트는 문제를 해결하거나 합리적인 의사결정에 기여함으로 가치를 창출합니다. 다른 데이터 직무처럼 프로그래밍과 같은 기술적인 역량이 요구될 뿐만 아니라, 상대적으로 더 높은 수준의 수학과 통계 지식이 요구되며, 몇몇 JD에서는 최신 논문을 읽고 업무에 적용할 수 있는 역량과 석사 이상의 학위를 요구하기도 합니다.

데이터 엔지니어

데이터 엔지니어는 다양한 기술을 활용하여 대규모의 데이터를 처리할 수 있는 데이터 파이프라인 구축을 담당하는 직무입니다. 이를 통하여 조직에서 데이터를 활용하는 (데이터 분석가, 데이터 사이언티스트를 비롯한) 인원들이 효율적으로 활용할 수 있도록 환경을 구축합니다. 다른 데이터 직무에 비해 컴퓨터 및 프로그래밍에 대한 역량을 많이 요구합니다.
